# agent-launch v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (inline) or superpowers:subagent-driven-development.

**Goal:** Single Bun binary that, given a `launch.yaml` and a project version, gathers repo context (CHANGELOG section, README, recent commits, optional release manifest) and emits 5 platform-native announcement drafts to `launches/v<version>/`. v0.1 is **draft-only** — no API posting.

**Architecture:** Bun + TS + Zod + Anthropic SDK. Flat `src/` (~5 files). Vendored per-platform prompt templates under `prompts/`. Drafts written as markdown with YAML frontmatter.

**Tech stack:** Bun, TypeScript strict, Zod, `js-yaml`, `@anthropic-ai/sdk` (model `claude-opus-4-7`).

**Refs:** [SPEC.md](../../../SPEC.md), [SCOPE.md](../../../SCOPE.md), [docs/superpowers/specs/2026-04-25-agent-launch-design.md](../specs/2026-04-25-agent-launch-design.md).

## Stage 2.1 — Context gathering + `context` command

### Task 1: Scaffold + git init

Files: `package.json`, `tsconfig.json`, `biome.json`, `.gitignore`, `LICENSE`, `CHANGELOG.md`, `launch.yaml.example`. Same shape as agent-fleet / agent-publish. Deps: `@anthropic-ai/sdk ^0.40.0`, `js-yaml`, `zod`.

- [ ] Init git, write files, `bun install`, `bun pm trust @biomejs/biome`, commit.

### Task 2: `src/config.ts` — `launch.yaml` Zod loader

- [ ] Test cases: valid 5-platform config, multiple `kind: reddit` allowed, invalid platform kind rejected, unknown subreddit format rejected, strict-mode unknown keys rejected, hooks array length 1–5 enforced.
- [ ] Schema (Zod):

```ts
const HnSchema = z.object({ kind: z.literal('hn'), pattern: z.enum(['show-hn', 'ask-hn', 'regular']) }).strict()
const RedditSchema = z.object({ kind: z.literal('reddit'), subreddit: z.string().regex(/^[a-zA-Z0-9_]{2,21}$/) }).strict()
const XSchema = z.object({ kind: z.literal('x'), handle: z.string() }).strict()
const MastodonSchema = z.object({ kind: z.literal('mastodon'), instance: z.string(), handle: z.string() }).strict()
const LinkedinSchema = z.object({ kind: z.literal('linkedin') }).strict()
const PlatformSchema = z.discriminatedUnion('kind', [HnSchema, RedditSchema, XSchema, MastodonSchema, LinkedinSchema])

const ProjectSchema = z.object({
  name: z.string().regex(/^[a-z0-9][a-z0-9-]*$/),
  oneliner: z.string().max(120),
  audience: z.string().max(200),
  hooks: z.array(z.string()).min(1).max(5),
}).strict()

const ContextSchema = z.object({
  repo: z.string().regex(/^[^/]+\/[^/]+$/),
  manifest: z.string().optional(),
}).strict()

const LaunchConfigSchema = z.object({
  version: z.literal(1),
  project: ProjectSchema,
  platforms: z.array(PlatformSchema).min(1),
  context: ContextSchema,
}).strict()
```

- [ ] Commit: `feat(config): launch.yaml loader with strict Zod schema`.

### Task 3: `src/context.ts` — gather repo context

Inputs: `cwd`, `version`. Outputs: `{ changelog, readme, commits, manifest? }`.

- [ ] Commit: `feat(context): gather CHANGELOG + README + recent commits + optional manifest`.

## Stage 2.2 — Draft generation + 5 platforms

### Task 5: Vendored prompt templates `prompts/{hn,reddit,x,mastodon,linkedin}.md`

Each file: YAML frontmatter (`platform`, `length_cap`, `length_cap_field`) + sections `# System prompt`, `# User prompt`, `# Anti-examples`.

- [ ] Write 5 files. Each ~40-60 lines. Voice tuned per platform.

Example for `prompts/hn.md`:

```markdown
---
platform: hn
length_cap: 2000
length_cap_field: body
title_cap: 80
---

# System prompt

You are writing a "Show HN" announcement for an open-source project. HN's voice is direct, technical, and skeptical. The crowd hates marketing speak.

Output a JSON object with exactly two fields: { "title": string, "body": string }.

# User prompt

Project: {{project.name}}
One-liner: {{project.oneliner}}
Audience: {{project.audience}}
Key technical hooks (use these, do not invent new ones):
{{project.hooks}}

This release ({{context.changelog}}):
{{context.changelog_body}}

Recent commits (signal of work):
{{context.recent_commits}}

README excerpt:
{{context.readme}}

Write the title (≤80 chars) and body (≤2000 chars). Title MUST start with "Show HN: ". Body should explain what the project does, why it exists, and what's new in this release. Link the GitHub repo at the end. No bullet points unless they're truly necessary.

# Anti-examples

NEVER use these phrases or patterns:
- "revolutionary" / "game-changing" / "next-generation"
- "We're excited to announce"
- "Today we're proud to share"
- AI-cliche em dash patterns ("X — and Y", "X. Y. Z.")
- Hyperbolic adjectives ("blazing fast", "lightweight")
- "leverages" (use "uses")
- "in this article we will discuss"
- Excessive emoji
```

- [ ] Commit: `feat(prompts): vendor 5 platform-native prompt templates`.

### Task 6: `src/platforms.ts` — prompt loader + per-platform metadata

- [ ] Load and parse `prompts/<kind>.md` (frontmatter + sections).
- [ ] Tests: load each of 5 templates, parse correctly.
- [ ] Commit: `feat(platforms): prompt template loader`.

### Task 7: `src/draft.ts` — Claude call + length cap retry

- [ ] Function `draftOne(platform, project, context, deps)`: returns `{ content, length, capped }`.
- [ ] `deps.anthropic.messages.create(...)` mocked in tests.
- [ ] Length-cap retry: if `length > cap` and retries < 2, regenerate with explicit "shorter, ≤ N chars" instruction. After 2 retries, return the over-length draft with `capped: false`.
- [ ] Per-platform output handling: HN returns JSON `{title, body}`; Reddit same; X returns thread separated by `---tweet---`; Mastodon/LinkedIn plain text.
- [ ] Tests: each platform path; retry triggers on over-length; stops after 2 retries.
- [ ] Commit: `feat(draft): Claude SDK call with length-cap retry`.

### Task 8: `src/index.ts` wire `draft` command

- [ ] `agent-launch draft <version> [--platforms hn,reddit,x] [--out launches/v<version>/]`
- [ ] For each configured platform (filtered by `--platforms`), call `draftOne`, write to `<out>/<kind>[-<discriminator>].md` with frontmatter.
- [ ] Print summary: "Drafted N posts for v<version> in <dir>".
- [ ] Commit: `feat(cli): draft command writing markdown drafts to launches/`.

## Stage 2.3 — Conformance C1-C5

- **C1 — best-effort determinism:** mock Anthropic to return scripted output; run draft twice with identical inputs; assert byte-equality.
- **C2 — length caps respected:** mock to return over-length first time, on-cap second time; assert final draft within cap.
- **C3 — repo URL in every draft:** assert each draft contains a github.com URL pointing to `{{repo}}`.
- **C4 — no secrets, only declared fields:** scan drafts for token-shaped strings.
- **C5 — launch.yaml strict:** invalid configs throw.

Each gets a `conformance/Cn-*.test.ts` file. Commit per file.

## Stage 2.4 — Security S1-S5

- **S1 — secrets:** ANTHROPIC_API_KEY env-only; never logged.
- **S2 — tool surface:** v0.1 has NO bash/shell tool to model. Verified by code inspection (no tool_use blocks in the SDK call); test assertion: the SDK call params object contains no `tools` field.
- **S3 — prompt injection:** prompts vendored; user-supplied strings (CHANGELOG/README) flow into the prompt verbatim — out of threat model since the user wrote them.
- **S4 — no network writes:** v0.1 calls only Anthropic API; intercept fetch; assert no posts to social platforms.
- **S5 — output sandboxing:** `--out` rejects absolute paths and `..` traversal.

Tests in `tests/security/`.

## Open considerations

- Determinism best-effort: temperature 0 + same prompt should produce ≤ 1% char diff. C1 accepts byte-equal as the strictest target; if flaky in real runs, soften.
